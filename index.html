Marble is an artistic programming language, with goals for accessibility.

Programming languages are cool and all, but we need to move beyond discussions about how code is formatted. This is why I'm putting spaces in variable names. This is why spaces and tabs are completely irrelevant concepts in the context of indentation. Ideally, there would be no formatting discussions necessary, but I am wondering if there will still be over things like whether ranges should be inclusive-inclusive, incluse-exclusive, or whatever.

Another goal is having tags for functions. I very much like function and operator overloading. They bring me joy. Hierarchical tags will serve purposes such as namespaces. Specificity tags will differentiate between versions of functions which should be identical for expected inputs. Specificity tags would be the name difference between sorting algorithms, or could mark a function as memory safe, or could mark different types of lists and differentiate between non-resizable arrays and resizable vectors.

Another goal is accessibility for programmers who cannot see or cannot see well. Since spaces will be in variable names, there should be no problems when voicing identifiers. The Marble IDE will be self-voiced. I'm still not sure how to voice all that punctuation, but I will figure it out.
